# Mibu 專案總覽 - NotebookLM 心智圖資料

## 專案定位

Mibu 是一個「旅遊發現與本地連結平台」，透過遊戲化機制（扭蛋）幫助旅客發掘景點，同時連結當地商家和旅遊專家。

核心價值：旅遊版的扭蛋社群平台

---

# 一、五大頁面功能拆解

## 1. 登入頁 (Login)

### 功能清單
- Apple Sign-In 登入
- Google Native 登入
- OAuth 統一認證
- 多身份綁定（同一用戶可綁 Apple + Google）
- Token 管理（7天有效期）

### 設計理念
- 降低註冊門檻：一鍵 OAuth 登入
- 帳號安全：多身份綁定防止帳號遺失
- 跨平台一致：iOS/Android/Web 統一流程

### 技術實現
- API: POST /api/auth/mobile
- Token 儲存: iOS 用 SecureStore，Web 用 AsyncStorage

---

## 2. 首頁 (Home)

### 功能清單
- 用戶等級卡片（旅行新手 → 傳奇旅者）
- 每日任務入口
- 公告輪播區（announcement/flash_event/holiday_event）
- 活動標籤切換（所有活動/快閃活動/在地活動）
- 推薦碼入口

### 設計理念
- 日活驅動：每日任務激勵用戶回訪
- 等級成就感：視覺化進度條提升黏著度
- 資訊集中：一頁看到所有重要動態

### 商業邏輯
- 等級系統 → 解鎖功能（Lv.7 可投票）
- 任務獎勵 → XP 累積 → 等級提升
- 公告推送 → 活動曝光 → 商家導流

---

## 3. 扭蛋頁 (Gacha)

### 功能清單
- 國家/城市選擇器
- 扭蛋次數調整（1-15次，預設7）
- AI 行程生成（1-2分鐘處理）
- 優惠券獲得動畫
- 稀有度顯示（N/R/SR/SSR/SP）
- 剩餘額度提示

### 設計理念
- 遊戲化核心：扭蛋機制製造驚喜感
- 稀缺性設計：每日36次限制，鼓勵每日回訪
- AI 差異化：不是隨機抽，而是 AI 根據偏好生成

### 商業邏輯
- 每日限額 36 次 → DAU 驅動
- 優惠券稀有度 → 商家定價依據
- 背包容量 30 格 → 製造消費動力

### 關鍵指標
- 每日扭蛋次數: 36 次上限
- 單次抽取: 1-15 個景點
- 背包容量: 30 格

---

## 4. 圖鑑頁 (Collection)

### 功能清單
- 景點卡片瀏覽
- 城市/稀有度篩選
- 已讀/未讀標記（紅點提示）
- 景點詳情彈窗
- 加入最愛功能
- 加入黑名單功能
- 社群投票（排除/保留）

### 設計理念
- 收藏成就感：圖鑑完成度激勵探索
- 個人化管理：最愛/黑名單客製化體驗
- 民主化內容：用戶投票決定景點存廢

### 商業邏輯
- 圖鑑完成度 → 成就徽章 → 用戶黏著
- 最愛功能 → 用戶偏好數據 → 精準推薦
- 投票機制 → UGC 內容品質控管

---

## 5. 設定頁 (Settings)

### 功能清單
- 個人資料編輯（頭像、暱稱）
- 語言設定（繁中/英/日/韓）
- 帳號綁定管理
- 推播通知開關
- 最愛/黑名單管理入口
- 緊急聯絡人設定
- 登出/刪除帳號

### 設計理念
- 用戶控制權：完整的隱私與偏好設定
- 多語言支援：拓展國際市場
- 安全機制：緊急聯絡人、SOS 求救

# 二、四大用戶角色

## 1. 旅客 (Traveler)
- 權限: 扭蛋、圖鑑、行程規劃、聊天
- 狀態: 立即可用
- 核心價值: 發現景點、收集優惠券、享受服務

## 2. 商家 (Merchant)
- 權限: 優惠券發行、地點管理、交易核銷、數據分析
- 狀態: 需審核通過
- 核心價值: 獲取客流、優惠券導客、ROI 追蹤

## 3. 專員 (Specialist)
- 權限: 旅客追蹤、聊天服務、行程規劃協作
- 狀態: 需審核通過
- 核心價值: 提供專業導覽、賺取服務費

## 4. 管理員 (Admin)
- 權限: 公告管理、黑名單管理、系統設定
- 狀態: 超管限定
- 核心價值: 平台營運、內容審核

---

# 三、商業模式

## 收入來源

### 1. 商家積分系統（主要收入）
- 機制: 商家購買積分 → 發行優惠券
- 模式: SaaS 訂閱制

### 2. 優惠券核銷手續費
- 機制: 旅客核銷優惠券 → 平台抽成
- 模式: 交易抽佣

### 3. 推薦系統佣金
- 機制: 用戶邀請好友 → 獎勵可提現 → 平台抽佣
- 模式: 拉新分潤

### 4. 眾籌 IAP 分成
- 機制: 用戶透過 IAP 參與募資
- 模式: Apple/Google 分成後的平台收入

### 5. 專家服務費
- 機制: Specialist 提供付費導覽服務
- 模式: 服務費抽成

---

## 價值鏈

```
旅客（需求端）
    ↓ 扭蛋抽獎
優惠券/景點
    ↓ 核銷使用
商家（供給端）
    ↓ 服務協作
Specialist（服務端）
    ↓ 內容回饋
社群投票
    ↓ SEO 變現
官網（流量端）
```

# 四、核心模組設計理念

## Gacha Module（扭蛋模組）

### 設計目標
遊戲化激勵旅客探索，製造驚喜與成就感

### 核心機制
- 稀有度分級: N(32%) → R → SR → SSR(8%) → SP(2%)
- 每日限額: 36 次（製造稀缺性）
- AI 生成: 根據地區+用戶偏好生成行程

### 商業價值
- 日活驅動: 每日限額鼓勵回訪
- 商家定價: 稀有度決定優惠券價值
- 差異化: AI 生成而非隨機抽取

---

## Trip Planner Module（行程規劃模組）

### 設計目標
連結旅客與認證旅遊專家，提供客製化服務

### 核心機制
- 地圖定位: 實時位置與景點距離
- 專家配對: 根據地區/專長配對 Specialist
- 即時通訊: 旅客與專家聊天協作
- 位置追蹤: Specialist 可追蹤旅客（需授權）

### 商業價值
- 服務變現: Specialist 服務費抽成
- 信任建立: 專家認證提升平台公信力
- 安全保障: SOS 緊急求救功能

---

## Economy Module（經濟模組）

### 設計目標
長期用戶黏著，建立成長體系

### 核心機制
- 等級系統: 1-50+ 級，累積 XP 升級
- 成就徽章: 完成特定任務獲得徽章
- 每日任務: 連續登入獎勵加成
- 功能解鎖: Lv.7 可投票，Lv.10 可建議景點

### 商業價值
- 留存驅動: 等級進度條激勵持續使用
- 社群品質: 高等級用戶才能參與治理
- 數據累積: 用戶行為數據越來越完整

---

## Referral Module（推薦模組）

### 設計目標
用戶拉新，降低獲客成本

### 核心機制
- 推薦碼生成: 每用戶獨立推薦碼
- 獎勵梯級: 1人/3人/5人/10人里程碑獎勵
- 排行榜: 週榜/月榜激勵競爭
- 可提現: 獎勵可轉換為現金

### 商業價值
- 低成本獲客: 用戶帶用戶
- 病毒傳播: 獎勵機制激勵分享
- 社群效應: 排行榜製造競爭氛圍

# 八、風險與機會

## 風險
- API 不一致性：前端調用的部分 API 後端不存在
- 扭蛋等待時間：AI 處理 1-2 分鐘可能流失用戶
- 商家端遲緩：供給側不足影響用戶體驗
- 隱私疑慮：Specialist 追蹤需要明確授權

## 機會
- 遊戲化差異化：扭蛋機制獨特
- 多邊市場效應：旅客/商家/專家互補
- 社群內容：用戶生成內容降低營運成本
- 跨境擴展：多語言支援利於國際化
